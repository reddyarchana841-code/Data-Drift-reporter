How it works
  1. Upload a CSV file representing a snapshot of a business table.
  2. The system validates the schema and stores the file.
  3. Row count, null %, mean/median/std, and unique value counts are computed automatically.
  4. If a previous snapshot exists, the system compares it to the new one and computes a drift score.
  5. An AI narration module generates a business-friendly summary of any changes.

Tip: Try uploading sample_data/orders_week1.csv first, then re-upload sample_data/orders_week2.csv with the same dataset name (or use "Generate New Snapshot" on the dataset detail page) to see drift detection in action.
